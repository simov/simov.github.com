Express Admin

MySQL admin for NodeJS.

I was using Django to administrate a few MySQL databases and since I found NodeJS I thought it would be cool if I have my admins written in Javascript.

I didn't found any solution at that time and eventually I ended up writing my own simple script.

Of course I can't write a framework, an ORM and etc. all by myself and expect it to be competitive and maintainable. Therefore my requirements were to write a simple and maintainable solution.

You can checkout the documentation page. I'm constantly trying to improve it as well as the examples so the most up to date information should be found there.

Styler

Chrome styling extension.

I was looking for an easy way to reload my pages on file changed while I'm developing styles for injection into web sites (like the monokai theme for GitHub's code view). And while there are plenty of chrome extensions for injecting styles into pages I didn't found one that allow me to auto refresh my pages while developing them in Sublime Text.

Anyway I was somewhat familiar with chrome's extension development basics and I was pretty sure that I can hook up the os specific stuff pretty fast using nodejs and web sockets.

Lastly the project doesn't have any tests at the moment but alot of stuff is documented into a trello board so hopefully when the tests are ready they will cover most of the bugs.

recursive-fs

Asynchronous recursive file system operations.

The only place where the recursion occur is in the read module and I use an array's length to determine the bottom of the recursion. That's all.

There are some additional modules in the lib folder as well. They all rely on the read module.

sr-json

JSON - parser, render and serializer.

I needed an html editor for json.

So I figured out what json object and template I need in order to make a recursion in mustache.

The next thing was to actually generate those template parameters.

Lastly I serialized the html structure back to a valid json.

deep-copy

Deep copy any object or array.

This one I wrote after the sr-json module. They are very similar.

I needed only a deep copy for one of my projects, not an entire library with lots of other stuff that I won't use. The code ended up being pretty neat.

Partials support in consolidate.js

Really simple solution with some drawbacks as well. The partial paths are relative to their parent template and the partials object is modified after the first render.

But it took really long time to be approved by TJ and then lots of people use this solution so I guess it's worth the tradeoffs.
